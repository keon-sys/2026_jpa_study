# Chapter 7
## 상속 관계 맵핑
- 조인 전략
    - 엔티티마다 별도의 테이블을 구성한다.
    - 조회할때마다 조인이 많이 사용되므로 성능이 저하될 수 있다.
- 단일 테이블 전략
    - 모든 엔티티를 하나의 테이블에 저장한다.
    - 상속 관계를 구분하기 위해 타입 컬럼을 사용한다.
    - 다른 엔티티에서 사용하는 컬럼에 null이 포함될 수 있다.
    - 조회시 조인을 사용하지 않아 성능이 좋다.
- 하이브리드(구현 클래스마다 테이블 전략)
    - 조인 전략과 단일 테이블 전략의 장점을 모두 사용한다.
    - 각 구현 클래스마다 별도의 테이블을 구성한다.
    - 여러 자식 테이블을 함께 조회할 때 성능이 느리다.

## @MappedSuperclass
- 위의 상속 관계 맵핑은 부모 클래스가 테이블을 항상 가져야 한다는 단점이 있다.
- @MappedSuperclass는 등록일자 수정일자, 등록자, 수정자와 같은 여러 엔티이에서 공통으로 사용하는 속성을 효과적으로 관리할 수 있다.

## 복합 키와 식별 관계 맵핑
- 식별 관계
  - 부모의 PK + 본인의 PK를 합쳐서 복합 키로 사용
- 비식별 관계
  - 필수적 비식별 관계 : 외래키 null을 허용하지 않는다.
  - 선택적 필수적 비식별 관계 : 외래키 null을 허용한다.
  - @IdClass
  - @Embeddable
